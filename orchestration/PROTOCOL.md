# 隔离角色与研究门禁协议

本项目采用 Codex 主代理监督、隔离上下文子代理执行的工作方式。它借用多代理工作流的“有界输入—独立复核—门禁放行”原则，但不启动常驻代理、定时心跳或外部邮箱服务。Git 提交、状态更新和门禁裁决由主代理串行完成。

## 权限链

潘尧是最终决策者；Codex 主代理负责证据整合、复现实验、门禁和 Git 同步；设计者、攻击者和 Reviewer 只处理被点名的输入并写入指定文件。任何子角色都不得自行推进阶段、修改项目规约、改写他人产物或执行 Git 操作。

## 角色输入边界

| 角色 | 可读取 | 不得读取 | 唯一输出 |
|---|---|---|---|
| model_only设计者 | 规约、污染物问题陈述、冻结证据表 | BMDL快照、审计、匹配结果及辅助方案 | `research/bmdl/paired/model_only_*.md` |
| bmdl_assisted设计者 | 上述输入及对应的最小BMDL配对包 | model_only方案、其他候选完整BMDL数据 | `research/bmdl/paired/bmdl_assisted_*.md` |
| 盲评Reviewer | 匿名化的A/B方案、统一量表、证据表 | A/B身份映射、设计者历史、BMDL审计结论 | `research/bmdl/paired/BLIND_REVIEW.md` |
| 深入设计者 | 入围污染物、证据边界、已裁决的BMDL策略 | 其他角色的隐藏推理、后续攻击意见 | 当前轮 `design.md` |
| 攻击者 | 当前方案、证据台账、攻击清单 | 设计者隐藏推理、未来修订稿 | 当前轮 `attack.md` |
| 裁决Reviewer | 当前方案、证据台账、攻击意见、评分表 | 完整历史链和角色身份 | 当前轮 `review.md` |

## 现行门禁

历史 `G1` 至 `G19` 保留审计含义，不再限定当前方案数量。新方案按以下门禁逐案运行：

1. `A0 pollutant admission`：环境/健康意义、二级出水相关性和选择性障碍成立，问题陈述在原型检索前冻结。
2. `A1 biomimetic story`：存在实际生物吸附、富集、隔离、屏障、分配、选择性传输过程或直接同构的天然功能；污染物靶蛋白结合不能单独通过。
3. `A2 material science`：确切非蛋白材料实体、水相化学、匹配对照、选择性机制和先例攻击完成。
4. `A3 engineering release`：E1或E2所需SOP、质量账本、分析、二级出水/床层矩阵和停止阈值完整。
5. `A4 adversarial review`：至少一次隔离攻击和独立Reviewer裁决，无未解决critical/high。
6. `A5 rolling delivery`：方案单独交付潘尧审阅，不等待固定批次、主备组合或数量目标。
7. `A6 continuity`：状态一致、凭据扫描通过、远端已推送、全新克隆可恢复唯一下一动作。

每个门禁必须由主代理独立检查产物和校验结果，并在 `orchestration/gates/` 写入记录后，才能进入下一阶段。门禁记录只保存可审计的结论和证据，不保存模型隐藏推理。

## 失败行为

- 证据不支持：降级主张、补证或否决，禁止编造补齐。
- BMDL误导：记入严重关联计数，并按 `research_contract.yaml` 降级或排除。
- 任一三大硬门失败或仍有critical/high问题：修订、降为对照或淘汰，不以总分补偿。
- 多个角色修改同一文件：停止合并，由主代理从各自独立产物重新整合。
- 需要改变主场景、三大硬门或实验释放边界：升级给潘尧决定。
