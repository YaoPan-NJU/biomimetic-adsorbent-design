# BMDL 客观配对评价与最终使用策略

## 结论

BMDL 不进入本项目的创意设计阶段。后续设计者不得读取 BMDL 快照、匹配结果或辅助方案。独立方案冻结后，主代理可用清洗快照做两件事：检查是否遗漏常规材料对照，检查是否出现库中已经暴露的错误跨污染物联想。BMDL 不作为生物原型来源、证据权威或候选排序器。

这一裁决不是因为 BMDL 没有任何信息，而是因为它在受控试验中没有达到设计辅助门槛，同时触发了“至少两条严重错误关联”的排除条件。

## 数据审计

- 数据库连接和导出同时启用了 `default_transaction_read_only=on` 与 `readonly=True`。
- 快照包含48个生物原型、44个污染物画像、130条匹配和216个别名。
- `performance_data` 原始3015条在去除代理ID后只剩1076条唯一记录，1939条为重复，重复率64.3%。3015不得解释为证据量或用于性能排名。
- 130条匹配中仅46条标记为直接证据。49条设计转译中39条是模型推断。
- 有机污染物匹配频繁收敛到壳聚糖、PDA、植物单宁和硅藻，容易把普通材料家族误当作选择性生物原型。

## 试验1为何无效

首次匿名比较表面显示辅助侧十项全胜、平均提高5.8分。时间线复核发现，纯模型组开始生成时关键证据尚未完成统一点检，辅助组使用了更新后的证据台账。Reviewer明确把证据状态和边界修正列为主要胜因。因此首次结果存在输入版本混杂，已在 `PAIRED_EVALUATION_INCIDENT.md` 中否决，未用于政策裁决。

## 受控试验2

第二次试验把 `SOUL.md`、合同、短名单、证据台账和关键主张复核文件的SHA-256写入冻结清单。两臂均由全新隔离上下文生成、禁止联网；唯一差别是辅助臂可读取十个最小BMDL包。Reviewer只读取去身份正文和统一量表。

解盲映射为：A-U和B-X属于纯模型组，A-V和B-W属于辅助组。

| 指标 | 纯模型组 | 辅助组 | 辅助减纯模型 |
|---|---:|---:|---:|
| 胜场 | 2/10 | 8/10 | +6 |
| 平均总分 | 84.1 | 86.4 | +2.3 |
| 因果闭环 | 17.7 | 18.3 | +0.6 |
| 证据质量 | 9.2 | 9.3 | +0.1 |
| 原创性 | 11.4 | 12.3 | +0.9 |
| 原型家族数 | 8 | 8 | 1.00 |
| 进入正文的严重误导关联 | 0 | 0 | 0 |

辅助组达到至少6胜、因果和证据维度不下降、原创性不下降及多样性不下降，但平均增益只有2.3，未达到5分的设计辅助门槛。0.1分证据增益也不能归因于库提供了更强来源，因为辅助组正文的生物原型仍来自同一冻结证据；库内匹配主要被设计者当作错误或普通对照排除。

## 严重错误关联

辅助方案正文没有严重误导，是因为设计者主动拒绝了错误匹配，并不说明输入本身安全。最小包至少暴露以下问题：

1. PFOA画像含“芳香环”和“大分子”等错误特征。
2. 磷酸盐匹配到实际针对As(III)施氏矿物的记录，并标作直接证据。
3. As(III)匹配到内容仅为通用相互作用的丝素记录，却标作直接证据。
4. SMX、BPA、硝酸盐和PFOS多次收到从染料、Cr(VI)或重金属迁移来的机制。

因此，合同中的“严重错误关联至少2条”排除条件被触发。设计者能够识别错误，不等于这些错误可以安全地暴露给创意生成阶段。

## 第一性原理评价

BMDL真正增加的是一个有限的检索索引，而不是可靠的机制推理层。它对已有精确画像的目标可以提示通用材料和常规吸附路径；对克拉霉素和ODV没有画像，对GenX有画像但无原型匹配。对于需要“生物特征→材料实现→动态因果”的任务，决定质量的是直接结构证据和对照设计。BMDL目前既没有覆盖这些关键目标，也不能稳定区分目标机制与跨污染物类比。

最合理的用途是后验负面审计：把壳聚糖/PDA等常规路线加入基准，把错误联想加入攻击清单。若未来库完成来源级修复、画像校验和直接证据重建，可重新开展同输入配对试验；在此之前不重新开放设计权限。
