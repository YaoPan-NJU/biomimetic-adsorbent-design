# Adversarial record protocol

Each concept uses a separate directory. Every round contains only role deliverables and adjudication records:

- `design.md`: current falsifiable proposal;
- `attack.md`: claim-specific challenges with severity and evidence;
- `review.yaml`: dispositions, scores, unresolved issues, and next action;
- `revision.md`: differences applied to the current proposal.

Do not store hidden chain-of-thought. Later roles receive the current design, evidence ledger, rubric, and unresolved issue list. They do not receive private reasoning or the full conversation transcript.
