# 千方案计划规约（fresh_1000）

## 目标

潘尧于 2026-07-19 指定 20 种污染物，每种目标 50 个仿生设计方案，共计 1000 个方案槽。每个方案必须是可支撑高水平论文发表的完整闭环。**数量为探索上限，质量为硬约束，禁止凑数。**

## 推进策略（2026-07-21 潘尧修订）

每个污染物的最低目标为至少一个通过方案（≥85 分、零未决 critical/high、创新性清单通过）。某污染物取得首个通过方案后，即推进到执行顺序中的下一个污染物（广度优先）；20 个污染物全部访问一轮后，再回溯为各污染物补充额外角度与槽位（每污染物上限 50 槽）。当前设计跑完后再切换，不打断运行中的槽。

## 单方案闭环标准

每个方案文件（`<slug>/SCHEMES/S<nn>_<简称>.md`）必须包含：

1. 科学问题与工程难题：继承污染物简报，并在方案层面具体化；
2. 生物原型与证据：一手文献为主，每条证据含 DOI/URL、标题、年份、支持的主张、定位（图表/页节）、证据层级、核验状态；
3. 仿生对应矩阵：至少 1 条硬对应（生物特征 → 来源支持 → 材料实现 → 预期功能 → 测量 → 因果对照）；动态对应可选，若声称须独立满足同一标准；
4. 材料架构与选择性机制：人工识别单元（非蛋白、非折叠肽），载体类别不预设；报告口径为总干复合体 mg/g 与 packed-bed mg/mL，附可及位点密度与空白载体贡献；
5. 创新性检查清单逐项结论（`INNOVATION_CHECKLIST.md` A-E 五维）；
6. 台架合成 SOP 草图；
7. 实验与因果对照计划：真实竞争物与结构近似物、二级出水或相应真实基质、再生、泄漏、稳定性、床层；
8. 成功与失败判据（可证伪）；
9. 攻击与裁决记录摘要、问题台账闭合情况、最终得分（合同 final_scoring 量表：因果闭环 20、选择性吸附机制 25、人工材料可转译性 20、原创性 15、实验可证伪与对照 10、证据完整性 10；通过线 85，且未决 critical/high 为 0）。

## 角色与流程

- 设计者（隔离上下文）：从指定角度产出完整方案，只读污染物简报、角度定义与证据边界；
- 攻击者（隔离上下文）：寻找致命缺陷（科学、化学、仿生真实性、选择性、证据、先例、工程）；
- 裁决者（隔离上下文）：按量表打分，出问题台账与创新性检查结论；
- 监督者（Claude Code 主代理）：证据控制、状态文件、Git 提交与推送、门禁裁决，串行完成。

迭代政策：每个方案槽执行两轮设计-攻击-裁决；未通过则带问题台账换角度推倒重建，单个槽至多 3 次重建；重建仍不通过则终止该槽并记录原因。任何轮次不降低评分门槛。

## 耗尽条款

若某污染物的机制可区分角度全部用尽，且重建仍无法通过，如实记录该污染物实际通过数 N（可小于 50），附角度耗尽清单与终止原因汇总。诚实的 N<50 优于凑数的 50；此条款是"禁止凑数"指令的执行形式。

## 节奏与推送

- 每个污染物先完成简报与角度地图（Phase A：环境存在、难去除机制与选择性难题结构、先例系统检索、原型空间三线扫描、简报综合），再进入 Phase B；
- Phase B 以单个方案槽为单位执行（每槽一个独立工作流，设计-攻击-裁决串行，约 15-25 分钟）：各角色把全文产物写入 `SCHEMES/_wip/` 指定文件、只返回结构化摘要；槽完成后监督者从 `_wip/` 组装正式方案文件，立即提交并推送，并在 `STATUS.yaml` 记录逐槽状态；会话中断后从 `STATUS.yaml` 与 `_wip/` 恢复，禁止整批作为单一工作流执行（单点失败会丢失整批）；脚本渲染规则见 `OPS_NOTES.md`；
- 每完成一个方案（通过或终止）：单独提交并推送 `origin/Qwen`；
- `STATUS.yaml` 每批更新；`HANDOFF.md` 与 `PROJECT_STATE.yaml` 每个污染物完成时更新；
- 证据台账 `research/evidence/evidence_ledger.csv` 按批追加核验条目。

## 水质场景与证据规则

水质场景按污染物论证（市政二级出水、工业废水、地表水、饮用水源、地下水），简报给出真实浓度区间与暴露场景；机制与因果对照用超纯水。证据规则遵循 `SOUL.md`：仅阅读摘要不得标 verified；数据库文本不是独立验证；禁止编造引用、参数与机制；事实、线索、推断与设计假设分层标注。

## 历史教训引用

历史产物（portfolio_20、portfolio_100、top5 各轮、S11、D1-A/D1-B、S1-SYN 等）为审计证据与反例库，不继承其分数与选择。各污染物简报设"已知陷阱"节，注入与该污染物相关的历史失败教训，供设计者与攻击者引用。

## 两条可选仿生启发思路

`research/evidence/BIOLOGICAL_TREATMENT_UNIT_BIOMIMETIC_HEURISTIC.md`（生物处理单元为原型）与 `research/evidence/ORGAN_ENRICHMENT_BIOMIMETIC_HEURISTIC.md`（组织/器官富集为原型）均为可选、非评分的发现启发式。Phase A 的原型扫描对每个污染物分别评估两条思路是否够格；不够格则标注不适用，禁止强行凑用。
